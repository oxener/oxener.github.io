Frank Oxener Resumé - experience

Sensecap - 2020-2021

Sensecap is an Internet of Things service for monitoring an experimental oyster mushrooms farm, where the mushrooms are cultivated on used coffee grounds. It uses several LoRa sensors (for temparature, humidity, CO2, soil moisture etc) and it provides an production workflow in order to correlate the measurements with the production batches.

Sensecap is a Elixir/Phoenix application and uses LiveView, DID.app (for passwordless authentication) and an integration with thethingsnetwork.org (TTN). It is running in a Docker container with a PostgreSQL database (deployed with Cloud66).

Consultify.app - 2020-2021

Consultify uses systemic consensing to measure the least amount of resistance in a group or community. It is based on the so called SK-prinzip developed by Erich Visotschnig and Siegfried Schrotta.

Consultify is a Elixir/Phoenix application and uses LiveView and DID.app (for passwordless authentication). It is running in a Docker container with a PostgreSQL database (deployed with Cloud66).

Rompslomp B.V. - 2019

Maintenance of the full stack Rails Rompslomp application. With Rompslomp accounting is made easy for SMB and freelancers.

Also responsible for the roll out of new features, such as for example two-factor authentication, external API with OAuth authetication and swagger documentation.

The Rompslomp application is running on Amazon EC2 with a MySQL database on RDS with the use of Cloud66 for provisioning and deploying.

I2C Sensor toolkit - 2018

Development of a toolkit with the Elixir/Nerves framework in order to make use of I2C environmental sensors on the Raspberry Pi platform, like for example temperature, air pressure, humidity, CO2, soil moisture etc. Monitoring data was collected with the use of InfluxDB.

Medangel - 2017-2019

Maintenance of the Phoenix backend for monitoring temperature of medicines.

MedAngel ONE is a solution that helps you to keep your medications at the right temperature. Measurements are done with a wireless smart sensor in combination with a mobile application.

The Phoenix application is running in a Docker container on Amazon EC2 with a PostgreSQL database on RDS and using Cloudwatch for logging.

KDVnet.nl - 2009-2017

As the single developer, I was responsible for the development, maintenance and hosting of a full-featured Rails system for childcare centers for the primary administrative workflow, such as planning, contracting, invoicing, debtor management etc. Since 2012 I had to manage a growth of 30-40% every year in traffic (and revenue). What started as a single server application became a full 'cluster' including load balancing, application servers, background servers, fully replicated database servers (PostgreSQL, Redis) and monitoring.

In the last year, the decision was made to migrate to a Phoenix/Elm stack. The first step was to create a hybrid application by session sharing with a Phoenix backend. The Phoenix backend was responsible for handling the more intensive background jobs in order to reduce the load on the Rails backend.

During this period the mt940 parser implemented for Dutch banks was open sourced, see mt940
